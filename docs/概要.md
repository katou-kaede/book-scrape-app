## 全体構成
- Scraper (Go + Playwright): 定期的にサイトを巡回し、SQLiteにデータを保存。
- Backend API (Go): SQLiteからデータを取得し、JSON形式で提供。
- Frontend (React): APIからデータを取得して一覧表示。
- Database (SQLite): プロジェクト内のファイルとして保存。

## 2. アーキテクチャ
### A. Frontend (React)
- 役割: ユーザーインターフェースの提供。

- 主な処理:
    - ブラウザからバックエンドAPI（Echo）を叩いて最新の書籍リストを取得。
    - 取得したJSONデータをテーブル形式で綺麗に表示。

- Docker設定: 開発中はソースコードをマウントし、保存と同時に画面が更新される「ホットリロード」を有効にします。

### B. Backend (Go + Echo)
- 役割: APIサーバーとしての司令塔。

- 主な処理:
    - /api/books エンドポイントを提供し、SQLiteからデータを抽出してReactへ渡す。
    - CORS（異なるポート間通信）の設定を行い、Reactからのリクエストを許可する。
    - （必要に応じて）スクレイピング実行のトリガーを引く。

### C. Scraper (Go + Playwright)
- 役割: 外部サイトからのデータ収集。

- 主な処理:
    - Headless Chromium（画面のないブラウザ）を立ち上げ、対象URLへアクセス。
    - DOM操作（CSSセレクタなど）を用いて、タイトル・価格・在庫状況を抽出。
    - 抽出したデータをSQLiteデータベースへ保存。

### D. Database (SQLite)
- 役割: データの永続化。

- 特徴:
    - ファイルベース（books.db）のため、別途データベースサーバーを立てる必要がなく軽量。
    - Dockerの**Volume（ボリューム）**機能を使ってホストOS側に保存することで、コンテナを停止・削除してもデータが消えないようにします。


## 3. データの流れ
1. 起動時: Docker Composeが各コンテナを立ち上げ、Goの init() または main() 内でSQLiteのテーブルが自動作成されます。

2. スクレイピング: Go側でPlaywrightが走り、books.toscrape.com からデータを取得し、books.db に INSERT（保存）します。

3. 表示リクエスト: ユーザーがReactの画面（localhost:3000）を開くと、ReactがEcho（localhost:8080）にデータ送信を要求します。

4. レスポンス: EchoがSQLiteからデータを読み込み、JSON形式でReactに返却、画面に反映されます。


## 4. バックエンド
`air`コマンドをインストールすることで、ホットリロードが可能になる。
`go install github.com/air-verse/air@latest`

初期ファイルを作成
```
cd backend
air init
```

`backend/.air.toml`を追加
```
root = "."
testdata_dir = "testdata"
tmp_dir = "tmp"

[build]
  # Windows用のビルドコマンド（cmd/api/main.go を指定）
  cmd = "go build -o ./tmp/main.exe ./cmd/api/main.go"
  # 実行するバイナリの場所
  bin = ".\\tmp\\main.exe"
  # 監視する拡張子
  include_ext = ["go", "tpl", "tmpl", "html", "toml"]
  # 無視するディレクトリ
  exclude_dir = ["assets", "tmp", "vendor", "frontend"]
  # ファイル変更からビルド開始までの待ち時間（ミリ秒）
  delay = 1000
  # ビルドエラー時に古いバイナリを動かし続けるか
  stop_on_error = true

[log]
  time = true

[color]
  main = "magenta"
  watcher = "cyan"
  build = "yellow"
  runner = "green"

[screen]
  clear_on_rebuild = true
```

## 5. フロントエンド
`Tailwind`を導入する。
```
cd frontend
npm install tailwindcss @tailwindcss/postcss postcss autoprefixer
```

`tailwind.config.js` を開き、Tailwind を適用するファイルを指定します。
```
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

`postcss.config.js`
```
export default {
  plugins: {
    '@tailwindcss/postcss': {},
    autoprefixer: {},
  },
}
```

`frontend/src/index.css` の中身を全部消して、以下にします。
```
@import "tailwindcss";
```

## 6. Docker
コンテナ立ち上げの際に、フロントエンドのビルドも同時に行うこと。

```powershell
# セミコロンで繋いで連続実行
cd frontend; npm run build; cd ..; docker compose up --build
```
